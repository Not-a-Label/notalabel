Introduction

Welcome to the Not a Label API. This RESTful API provides programmatic access to all features of the Not a Label platform, including AI-powered marketing tools, artist profiles, and analytics.

Base URL

https://not-a-label.art/api

Request Format

All requests must include the following headers:

Content-Type: application/json
Authorization: Bearer YOUR_JWT_TOKEN

Authentication

Not a Label uses JWT (JSON Web Tokens) for authentication. You must include your token in the Authorization header for all protected endpoints.

POST Register User

/auth/register

Create a new user account.

Request Body

Parameter Type Required Description
email string Yes User's email address
password string Yes User's password (min 6 characters)
name string Yes User's display name
artistType string No Type of artist (musician, producer, etc.)

Example Request

curl -X POST https://not-a-label.art/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "artist@example.com",
    "password": "securepassword",
    "name": "John Doe",
    "artistType": "musician"
  }'

Response

200 OK Success

{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": 1,
    "email": "artist@example.com",
    "name": "John Doe",
    "artistType": "musician"
  }
}

AI Assistant

POST AI Assistant Chat

/ai/assistant

Chat with the AI assistant for career advice and insights.

Request Body

Parameter Type Required Description
message string Yes Your message to the AI assistant
context object No Additional context for the conversation

Example Request

curl -X POST https://not-a-label.art/api/ai/assistant \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "How can I grow my fanbase on social media?",
    "context": {
      "genre": "indie rock",
      "experience": "2 years"
    }
  }'

Response

200 OK Success

{
  "response": "Growing your indie rock fanbase on social media requires a multi-faceted approach..."
}

POST Generate Marketing Content

/marketing/templates/generate

Generate AI-powered marketing content.

Request Body

Parameter Type Required Description
type string Yes Type of content (social_media, email, etc.)
context object Yes Context for content generation
tone string No Desired tone (professional, casual, excited)

Example Request

curl -X POST https://not-a-label.art/api/marketing/templates/generate \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "social_media",
    "context": {
      "artistName": "The Midnight Echoes",
      "genre": "Synthwave",
      "announcement": "New album release"
    },
    "tone": "excited"
  }'

POST Create Marketing Post

/marketing/posts/create

Create a new marketing post.

Request Body

Parameter Type Required Description
type string Yes Post type (announcement, promotion, engagement)
content string Yes Post content
platform string Yes Target platform (instagram, twitter, facebook, email)
scheduledFor datetime No Schedule post for future (ISO 8601 format)

Error Handling

The API uses standard HTTP response codes to indicate success or failure. Error responses include a descriptive message.

Common Error Codes

400 Bad Request Invalid request parameters

{
  "error": "Email already exists"
}

401 Unauthorized Missing or invalid authentication

{
  "error": "Invalid credentials"
}

403 Forbidden Insufficient permissions

{
  "error": "Access denied"
}

404 Not Found Resource not found

{
  "error": "Post not found"
}

500 Internal Server Error Server error

{
  "error": "Internal server error"
}

Complete Examples

Full Marketing Workflow

# 1. Register a new user
TOKEN=$(curl -s -X POST https://not-a-label.art/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "newartist@example.com",
    "password": "MySecurePass123",
    "name": "Rising Star",
    "artistType": "musician"
  }' | jq -r '.token')

# 2. Generate marketing content
CONTENT=$(curl -s -X POST https://not-a-label.art/api/marketing/templates/generate \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "social_media",
    "context": {
      "artistName": "Rising Star",
      "genre": "Pop",
      "announcement": "First single release"
    },
    "tone": "excited"
  }' | jq -r '.content')

# 3. Create a marketing post
curl -X POST https://not-a-label.art/api/marketing/posts/create \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{
    \"type\": \"announcement\",
    \"content\": \"$CONTENT\",
    \"platform\": \"instagram\",
    \"scheduledFor\": \"2025-06-01T18:00:00Z\"
  }"

# 4. Check your posts
curl -X GET https://not-a-label.art/api/marketing/posts \
  -H "Authorization: Bearer $TOKEN"
